# Ваш приказ

«Ваш приказ» — локальная пошаговая стратегия на квадратной клеточной карте. Игрок основывает замок, строит экономику и укрепления, нанимает отряды и уничтожает замки остальных участников.

Этот файл описывает только реализованное состояние проекта. Числовые правила игры находятся в `src/config/rules.ts` и `src/config/game.ts`.

## Технологии

- Vite, React 19 и TypeScript.
- Canvas 2D для карты, местности, объектов, маршрутов и эффектов.
- HTML/CSS для меню, HUD, панелей команд и модальных окон.
- Vitest для тестов и ESLint для статической проверки.
- Серверная часть отсутствует; данные хранятся в браузере.

Canvas отрисовывает видимую часть карты с учётом `devicePixelRatio`. Перерисовка запускается при изменении состояния, камеры, размера окна или во время короткой анимации.

## Начало партии

На главном экране доступны три встроенных сценария и сохранённые пользовательские карты. Число участников выбирается в диапазоне от 2 до 4: один участник является игроком, остальные — NPC.

Перед партией карта делится на связные стартовые владения. Расчёт учитывает площадь, леса, высоты, форму границ и наличие допустимых мест для замков. Каждое владение должно содержать не менее 24 свободных проходимых клеток, отдельную площадку 2 × 2 на равнине, площадку 2 × 2 на высоте и ещё одну свободную площадку 2 × 2. Эти площадки не пересекаются между собой и не используются для стартового замка. Если хотя бы одно владение не удовлетворяет условиям, карта считается непригодной для выбранного числа участников.

Игрок выбирает владение и ставит свой замок на разрешённую клетку. Замки NPC автоматически ставятся в остальных владениях.

Каждый участник начинает со следующими значениями:

| Параметр | Значение |
| --- | ---: |
| Дерево | 110 |
| Камень | 80 |
| Руда | 0 |
| Железо | 16 |
| Зерно | 36 |
| Мясо | 12 |
| Золото | 105 |
| Население | 5 |
| Предел населения | 5 |
| Обслуживание кухнями | 20 |
| Приказы на ход | 8 |

Замок имеет 100 прочности и производит 4 зерна и 2 золота за ход. Начальный режим налогов — умеренный.

## Карта и генератор

Карты всегда квадратные. Генератор поддерживает размеры от 50 × 50 до 150 × 150 клеток; размер по умолчанию — 100 × 100.

Клетка содержит местность и не более одного игрового объекта. Реализованы:

- равнина;
- проходимая высота;
- непроходимый пик;
- лес на равнине или высоте;
- замок, здание или отряд.

Лес проходим для отрядов, но удваивает стоимость входа в клетку. Он также блокирует линию огня лучников и нужен для размещения лесопилок на соседних свободных клетках. Лес не вырубается и не изменяется во время партии.

Генератор использует seed и настраиваемые доли высот, пиков и леса. Также настраиваются масштаб форм, распределение рельефа и растительности между центром и краями, предпочтительная высота леса и влияние рельефа. В гибридном и ручном режимах крупные формы задаются маркерами на сетке 12 × 12.

Границы владений рассчитываются отдельно от генерации местности в Web Worker. Повторные изменения настроек объединяются короткой задержкой, а после технического сбоя Worker расчёт автоматически повторяется один раз. При одинаковых настройках, seed и числе участников результат детерминирован; поиск прекращается на первом пригодном варианте.

Сохранённая пользовательская карта содержит параметры генератора и ручную сетку, а не готовую матрицу клеток. Эти данные хранятся в `localStorage`.

## Управление картой

- ЛКМ или СКМ с перетаскиванием перемещает камеру.
- Колесо масштабирует карту относительно позиции указателя.
- Масштаб ограничен диапазоном от 0.3 до 1.5; камера не выходит за границы карты.
- Обычный клик выбирает клетку или объект.
- ПКМ или `Ctrl + клик` открывает контекстное меню.
- Удержание `Shift` показывает владения и меняет курсор над картой на `grab`.
- `Esc` закрывает контекстное или модальное меню, отменяет активное строительство, найм или разделение и только затем открывает настройки.

Контекстное меню позволяет приказать выбранному отряду идти в свободную клетку, разделить или подготовить объединение отряда, а также удалить принадлежащий игроку объект, кроме замка.

Приказ «Идти сюда» строит самый дешёвый по приказам ортогональный путь в обход пиков и известных занятых клеток. Поэтому маршрут может обойти лес, если такой путь требует меньше приказов, и использует принадлежащие отряду барбаканы как проходы. Неизвестная занятая клетка не выдаёт себя при расчёте пути: при столкновении отряд переходит в бой. Отряд делает шаг каждые 300 мс, заново проверяет маршрут и останавливается у цели, при нехватке приказов или при блокировке пути.

## Обзор и туман войны

Во время партии клетки вне текущего обзора слегка затемняются, но рельеф, владения, замки, гражданские здания и укрепления остаются различимы. Туман скрывает только чужие отряды, гарнизоны и казармы. Скрытый объект также не отображается в выборе клетки, контекстном меню и маркерах действий.

- замок и каждое обычное здание открывают клетки в радиусе 8;
- башня открывает клетки в радиусе 12;
- отряд на равнине или в лесу открывает радиус 10;
- отряд на проходимой высоте открывает радиус 12.

Обзор рассчитывается как круглая область и обновляется после каждого изменения карты. Состав чужого гарнизона башни не раскрывается за пределами обзора.

## Ход

В начале каждого хода игрок получает 8 приказов. Остаток не переносится.

При наведении или клавиатурном фокусе на доступном действии его будущая стоимость показывается прямо на шкале приказов: соответствующие жёлтые маркеры временно становятся красными. После выбора строительства, найма или разделения прогноз остаётся видимым до выполнения либо отмены действия.

| Действие | Приказы |
| --- | ---: |
| Шаг отряда без рыцарей по клетке без леса | 1 |
| Шаг отряда без рыцарей по клетке леса | 2 |
| Шаг отряда с рыцарями по клетке без леса | 2 |
| Шаг отряда с рыцарями по клетке леса | 4 |
| Проход обычного отряда через свой барбакан | 2 |
| Проход отряда с рыцарями через свой барбакан | 4 |
| Ближняя или дальняя атака | 1 |
| Разделение отряда | 2 |
| Частичное увольнение бойцов | 2 |
| Объединение отрядов | 2 |
| Ввод лучников в башню или вывод из неё | 2 |
| Найм от 1 до 10 бойцов одного типа | 2 |
| Снос своего здания или расформирование отряда | 2 |
| Обычное строительство | 4 |
| Строительство казармы, церкви или башни | 6 |

Ход можно завершить с неизрасходованными приказами. После нажатия кнопки управления блокируются на одну секунду, затем экономика всех участников пересчитывается, номер хода увеличивается и игрок снова получает 8 приказов. NPC не строят, не нанимают войска, не перемещаются и не атакуют.

## Строительство

Строительство доступно только внутри стартового владения игрока. Все клетки постройки должны быть свободны, проходимы и соответствовать требованиям местности. Границы владений во время партии не изменяются.

«Свободная» местность в таблице означает равнину или высоту без леса.

| Постройка | Размер | Местность | Приказы | Стоимость | Эффект за ход | Прочность |
| --- | --- | --- | ---: | --- | --- | ---: |
| Ферма | 2 × 2 | равнина без леса | 4 | 28 дерева, 8 камня, 8 золота | +18 зерна при 2 работниках | 10 |

| Лесопилка | 1 × 1 | свободная проходимая клетка, ортогонально соседняя с лесом | 4 | 14 дерева, 5 камня, 5 золота | +16 дерева при 1 работнике | 15 |
| Каменоломня | 2 × 2 | высота без леса | 4 | 20 дерева, 8 золота | +12 камня при 2 работниках | 18 |
| Рудник | 1 × 1 | высота без леса | 4 | 18 дерева, 8 камня, 10 золота | +6 руды при 1 работнике | 15 |
| Плавильня | 2 × 2 | свободная | 4 | 32 дерева, 28 камня, 24 золота | до 5 руды в 5 железа при 2 работниках | 22 |
| Кухня | 1 × 1 | свободная | 4 | 20 дерева, 12 камня, 8 золота | обслуживает 20 жителей при 1 работнике | 14 |
| Дом | 1 × 1 | свободная, не дальше 5 клеток от своего замка или кухни | 4 | 25 дерева, 10 камня | +10 к пределу жилья | 10 |
| Казарма | 2 × 2 | свободная | 6 | 36 дерева, 30 камня, 18 золота | позволяет нанимать по всему периметру | 25 |
| Церковь | 2 × 2 | свободная | 6 | 20 дерева, 45 камня, 60 золота | +1 к приросту населения, содержание 4 золота | 30 |

Многоклеточная постройка считается одним объектом: производство, содержание, урон и снос применяются один раз ко всему зданию.

Барбакан занимает клетку ворот. Свой отряд может одним перемещением пройти с соседней клетки на свободную клетку с противоположной стороны, не занимая клетку здания. Такой проход оплачивается как два обычных шага. Чужой барбакан пройти нельзя: противник сначала разрушает его, после чего освободившаяся клетка становится обычным проходом для всех.

## Население, работники и снабжение

Мирное население делится на занятых работников и свободных жителей. Работники остаются частью мирного населения. Новый боец забирает одного мирного жителя и после найма учитывается отдельно в составе армии. Сумма мирных жителей и бойцов ограничена пределом населения.

Работники назначаются автоматически при расчёте экономики. Приоритет: фермы, охотничьи дома, кухни, лесопилки, каменоломни, рудники и плавильни. Внутри каждого типа здания обработка идёт сверху вниз и слева направо. Ферме, каменоломне и плавильне нужны по 2 работника; остальным перечисленным зданиям — по одному. При частичной занятости производство или переработка пропорционально уменьшается с округлением вниз. Назначения не хранятся отдельно и всегда вычисляются из текущего населения и построек.

Предел мирного населения определяется двумя независимыми значениями: доступным жильём и кухонным обслуживанием. Замок даёт 5 начальных мест жилья и обслуживает 20 жителей, каждый дом добавляет 10 мест жилья, а каждая работающая кухня — ещё 20 мест обслуживания. Фактический предел равен меньшему из этих значений. Поэтому первый дом позволяет вырасти до 15 жителей, второй — до 20, а дальнейший рост требует кухни.

Дом можно поставить не дальше 5 клеток по ортогональному расстоянию от своего замка или кухни. Разрушенный дом сразу перестаёт давать места. Разрушенная или остановившаяся из-за нехватки работника кухня перестаёт давать обслуживание. Если мирное население становится выше доступного предела после потери жилья или обслуживания, оно уменьшается не более чем на одного жителя за ход.

Потребность в пище за ход равна:

`ceil(мирное население × множитель налогов) + бойцы`

Множитель налогов равен 1 без налогов, 1.5 при умеренных и 2 при грабительских. Общая потребность покрывается зерном и мясом в пропорции, максимально близкой к равной. При нехватке одного вида пищи остаток покрывается другим.

Если вся потребность оплачена и все жители обслуживаются кухнями, мирное население увеличивается на 1. Разнообразное питание добавляет ещё 1 жителя, а каждая церковь — ещё 1 при условии, что содержание всех зданий и войск оплачено. Рост не превышает меньший из пределов жилья и кухонного обслуживания.

Если еды не хватает, владение теряет одного жителя. Сначала уменьшается число свободных жителей, затем динамическое распределение снимает работника с последнего здания. Если мирных жителей нет, погибает самый дешёвый доступный боец, включая гарнизоны: ополченец, затем копейщик, лучник и рыцарь; внутри одного типа выбор идёт детерминированно по координатам. Последний оставшийся житель или боец не погибает от голода.

Если не менее 25% общей потребности фактически оплачено зерном и не менее 25% — мясом, на следующий ход включается «Разнообразное питание». Бонус добавляет 1 к приросту мирного населения при наличии свободного жилья и увеличивает исходящий урон всех собственных отрядов на 5%. Он не суммируется и пересчитывается заново в конце каждого хода.

Содержание всех зданий и войск одного владения суммируется и округляется вверх до целого количества ресурса. Ресурсы не уходят ниже нуля. Если полного запаса на содержание не хватает, за ход дезертирует один самый дешёвый доступный боец, включая лучника из гарнизона.

Экономика завершения хода выполняется в фиксированном порядке: основное производство, начисление налогов, оплата содержания, переработка, снабжение пищей и изменение населения. Налоговое золото текущего хода может использоваться для содержания.

Плавильни работают при наличии руды и работников и не требуют содержания. Без налогов полностью укомплектованная плавильня перерабатывает не более 5 руды в 5 железа за ход; с одним работником — не более 2. Налоговый штраф уменьшает пропускную способность так же, как производство ресурсов. Лишняя руда остаётся в запасе.

## Налоги

Режим налогов меняется при выборе своего замка и не требует приказов.

| Режим | Золото | Потребность мирных жителей в пище | Производство и переработка зданий |
| --- | --- | --- | --- |
| Без налогов | 0 | ×1 | без изменения |
| Умеренные | `население` | ×1.5 | −1 к каждому производимому ресурсу и пропускной способности |
| Грабительские | `население × 2` | ×2 | −3 к каждому производимому ресурсу и пропускной способности |

Штраф налогов применяется к зданиям, но не к производству замка.

## Рынок

Торговля доступна при выборе своего рынка и не расходует приказы.

| Ресурс | Покупка | Продажа |
| --- | ---: | ---: |
| Дерево | 3 | 1 |
| Камень | 4 | 2 |
| Руда | 5 | 3 |
| Железо | 10 | 6 |
| Зерно | 2 | 1 |
| Мясо | 4 | 2 |

Цена указана в золоте за единицу ресурса.

Без налогов покупка 5 руды и продажа полученных 5 единиц железа оставляют 5 золота валовой маржи за ход переработки. Она требует заранее построенной плавильни, двух работников и ограничена её пропускной способностью. При умеренных и грабительских налогах пропускная способность ниже.

## Отряды

Отряд занимает одну клетку и содержит до 10 бойцов в любой комбинации; суммарный предел армии одного владения — 100 бойцов, включая гарнизоны башен. Найм возможен на проходимой клетке рядом с собственным замком или любой клеткой собственной казармы. Бойцов можно добавить в существующий отряд, если итоговая численность не превышает 10.

| Боец | Стоимость за одного | Урон | Стойкость | Золото за ход |
| --- | --- | ---: | ---: | ---: |
| Ополчение | 4 зерна, 8 золота | 1 | 1 | 0.5 |
| Копейщик | 4 зерна, 1 железо, 10 золота | 1.2 | 1.35 | 1 |
| Лучник | 4 дерева, 4 зерна, 16 золота | 1 | 1 | 1.25 |
| Рыцарь | 6 зерна, 2 мяса, 8 железа, 24 золота | 1.2 | 2.5 | 2.5 |

Разделение позволяет выбрать состав нового отряда, но не даёт оставить одну из частей пустой. Выбранные бойцы переносятся на соседнюю свободную клетку, а текущее здоровье делится пропорционально. Объединение переносит состав и здоровье в соседний собственный отряд при соблюдении лимита в 10 бойцов. Частичное увольнение возвращает выбранных бойцов в мирное население за 2 приказа без возврата ресурсов найма; полное расформирование отряда работает по тем же правилам.

## Бой

Ближний бой начинается при попытке отряда войти на соседнюю клетку с чужим отрядом, зданием или замком. Отряды обмениваются уроном одновременно. Высота увеличивает силу отряда на 12%, лес — на 8%; для клетки, одновременно являющейся высотой и лесом, применяется бонус высоты. «Разнообразное питание» дополнительно увеличивает исходящий урон отряда на 5%.

Обычный урон по стене умножается на 0.35. Башня и барбакан не получают такого сокращения урона; при прочности 20 барбакан является намеренно более слабой точкой оборонительной линии. Итоговый урон по постройке округляется вверх и составляет не менее 1. Если постройка разрушена в ближнем бою, атакующий отряд занимает её клетку.

Отряд с лучниками может атаковать по вертикали или горизонтали на расстоянии от 2 до 8 клеток. Пик, лес или объект между стрелками и целью блокирует выстрел. Высота стрелков увеличивает урон на 20%, лес на клетке цели уменьшает его на 25%. Дальняя атака не перемещает отряд.

В башню можно за 2 приказа ввести до 5 лучников из соседнего отряда и вывести гарнизон на соседнюю свободную клетку. Гарнизон стреляет по прямой на расстояние до 10 клеток за 1 приказ и получает постоянный бонус высоты 20%. Линию огня по-прежнему блокируют пики, лес и объекты. Башню с гарнизоном нельзя снести; при разрушении башни гарнизон погибает.

Партия получает статус победы после разрушения всех чужих замков. После победы карту можно продолжить осматривать, но игровые команды больше не выполняются.

## Сохранения и настройки

- Партии сохраняются отдельными слотами в IndexedDB и загружаются с главного экрана или из настроек. Перед загрузкой проверяется полная схема партии и числовые инварианты; сохранения несовместимой версии отклоняются. Запись считается успешной только после завершения транзакции IndexedDB.
- Пользовательские карты, язык, громкость и состояние обучающей подсказки хранятся в `localStorage`.
- Доступны русский и английский языки; словари загружаются раздельно.
- Громкость регулируется от 0 до 100. Значение 0 отключает звуки.

## Ограничения текущей версии

- NPC только владеют территориями, получают экономический пересчёт и автоматически пропускают ход.
- Захват клеток и изменение границ владений отсутствуют.
- Мультиплеер и серверное хранение отсутствуют.
- Управление рассчитано на desktop и мышь.

## Локальный запуск

Требуется Node.js `20.19+`; рекомендуемая версия для разработки — Node.js `22.13.1` (указана в `.nvmrc` и `.node-version`).

```bash
npm install
npm run dev
```

Проверки проекта:

```bash
npm run precommit
npm run build
```

`npm run precommit` последовательно запускает ESLint, проверку типов, тесты и production-сборку.
